# User Personas

**프로젝트**: 테이블오더 서비스  
**생성일**: 2026-02-25

---

## Persona 1: 고객 (Customer)

### 프로필
| 항목 | 내용 |
|-----|------|
| **이름** | 김민수 (대표 고객) |
| **역할** | 매장 방문 고객 |
| **기술 수준** | 일반 (스마트폰 사용 가능) |
| **사용 기기** | 테이블 태블릿 |

### 목표
- 빠르고 편리하게 음식 주문
- 직원 호출 없이 자율적으로 주문
- 주문 상태 확인

### 불편 사항 (Pain Points)
- 직원을 기다리는 시간
- 메뉴판을 보고 결정하기 어려움
- 주문이 제대로 들어갔는지 불안

### 사용 시나리오
1. 테이블에 앉으면 태블릿이 자동으로 메뉴 화면 표시
2. 메뉴를 탐색하고 장바구니에 담음
3. 주문 확정 후 주문 번호 확인
4. 추가 주문 시 같은 과정 반복
5. 주문 내역에서 현재까지 주문 확인

## Persona 2: 매장 관리자 (Admin)

### 프로필
| 항목 | 내용 |
|-----|------|
| **이름** | 박서연 (매장 매니저) |
| **역할** | 매장 운영 관리자 |
| **기술 수준** | 중급 (POS 시스템 사용 경험) |
| **사용 기기** | PC 또는 태블릿 |

### 목표
- 실시간으로 들어오는 주문 파악
- 주문 처리 상태 관리
- 테이블 회전율 관리

### 불편 사항 (Pain Points)
- 바쁜 시간대 주문 누락 우려
- 오래된 주문 놓치는 경우
- 테이블별 주문 현황 파악 어려움

### 사용 시나리오
1. 매장 오픈 시 관리자 화면 로그인
2. 대시보드에서 테이블별 주문 현황 모니터링
3. 신규 주문 확인 후 주방에 전달 (주문접수 → 조리중)
4. 음식 완성 시 완료 처리
5. 고객 퇴장 시 테이블 이용 완료 처리
6. 필요 시 과거 주문 내역 조회
