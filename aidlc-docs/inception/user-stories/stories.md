## Epic 1: 고객 주문 경험

### US-1.1: 메뉴 조회
**As a** 고객  
**I want to** 테이블 태블릿에서 메뉴를 조회하고 탐색할 수 있다  
**So that** 원하는 음식을 쉽게 찾아 주문할 수 있다

**Acceptance Criteria:**
- [ ] 태블릿 접근 시 메뉴 화면이 기본으로 표시된다
- [ ] 메뉴가 카테고리별로 분류되어 표시된다
- [ ] 각 메뉴에 이름, 가격, 설명, 이미지가 표시된다
- [ ] 카테고리 간 빠르게 이동할 수 있다
- [ ] 터치하기 쉬운 크기의 버튼/카드로 표시된다 (최소 44x44px)

---

### US-1.2: 장바구니 관리
**As a** 고객  
**I want to** 주문할 메뉴를 장바구니에 담고 수정할 수 있다  
**So that** 주문 전에 선택한 메뉴를 확인하고 조정할 수 있다

**Acceptance Criteria:**
- [ ] 메뉴를 장바구니에 추가할 수 있다
- [ ] 장바구니에서 메뉴를 삭제할 수 있다
- [ ] 메뉴 수량을 증가/감소할 수 있다
- [ ] 총 금액이 실시간으로 계산되어 표시된다
- [ ] 장바구니 전체를 비울 수 있다
- [ ] 페이지 새로고침 후에도 장바구니가 유지된다

---

### US-1.3: 주문 생성
**As a** 고객  
**I want to** 장바구니의 메뉴를 주문할 수 있다  
**So that** 음식을 받을 수 있다

**Acceptance Criteria:**
- [ ] 주문 전 최종 내역을 확인할 수 있다
- [ ] 주문 확정 버튼을 누르면 주문이 생성된다
- [ ] 주문 성공 시 주문 번호가 5초간 표시된다
- [ ] 주문 성공 후 장바구니가 자동으로 비워진다
- [ ] 주문 성공 후 메뉴 화면으로 자동 이동한다
- [ ] 주문 실패 시 에러 메시지가 표시되고 장바구니가 유지된다

---

### US-1.4: 주문 내역 조회
**As a** 고객  
**I want to** 현재 테이블에서 주문한 내역을 확인할 수 있다  
**So that** 무엇을 주문했는지, 주문 상태가 어떤지 알 수 있다

**Acceptance Criteria:**
- [ ] 현재 테이블 세션의 주문만 표시된다 (이전 고객 주문 제외)
- [ ] 주문 번호, 시각, 메뉴, 수량, 금액이 표시된다
- [ ] 주문 상태(주문접수/조리중/완료)가 표시된다
- [ ] 주문이 시간순으로 정렬된다

---

### US-1.5: 자동 로그인
**As a** 고객  
**I want to** 별도 로그인 없이 바로 메뉴를 볼 수 있다  
**So that** 즉시 주문을 시작할 수 있다

**Acceptance Criteria:**
- [ ] 태블릿이 초기 설정되면 이후 자동으로 로그인된다
- [ ] 자동 로그인 실패 시 초기 설정 화면이 표시된다
- [ ] 로그인 정보가 로컬에 안전하게 저장된다

---

## Epic 2: 관리자 주문 관리

### US-2.1: 관리자 로그인
**As a** 관리자  
**I want to** 매장 관리 시스템에 로그인할 수 있다  
**So that** 주문을 관리할 수 있다

**Acceptance Criteria:**
- [ ] 매장 식별자, 사용자명, 비밀번호로 로그인할 수 있다
- [ ] 로그인 성공 시 대시보드로 이동한다
- [ ] 세션이 16시간 유지된다
- [ ] 브라우저 새로고침 후에도 로그인이 유지된다
- [ ] 16시간 후 자동 로그아웃된다
- [ ] 로그인 시도 횟수가 제한된다

---

### US-2.2: 실시간 주문 모니터링
**As a** 관리자  
**I want to** 들어오는 주문을 실시간으로 확인할 수 있다  
**So that** 주문을 놓치지 않고 처리할 수 있다

**Acceptance Criteria:**
- [ ] 테이블별 카드 형태의 그리드 레이아웃으로 표시된다
- [ ] 각 테이블 카드에 총 주문액이 표시된다
- [ ] 각 테이블 카드에 주문 시간이 표시된다
- [ ] 신규 주문이 2초 이내에 화면에 표시된다 (SSE)
- [ ] 테이블 카드 클릭 시 전체 주문 상세를 볼 수 있다

---

### US-2.3: 주문 상태 변경
**As a** 관리자  
**I want to** 주문 상태를 변경할 수 있다  
**So that** 주문 처리 진행 상황을 관리할 수 있다

**Acceptance Criteria:**
- [ ] 주문접수/조리중/완료 상태를 버튼으로 선택할 수 있다
- [ ] 상태 변경이 즉시 반영된다
- [ ] 상태 변경이 고객 화면에도 반영된다

---

### US-2.4: 지연 주문 확인
**As a** 관리자  
**I want to** 오래된 주문을 쉽게 식별할 수 있다  
**So that** 지연된 주문을 우선 처리할 수 있다

**Acceptance Criteria:**
- [ ] 주문 후 30분이 경과한 주문은 카드 색상이 다르게 표시된다
- [ ] 지연 주문이 시각적으로 눈에 띄게 강조된다

---

### US-2.5: 주문 삭제
**As a** 관리자  
**I want to** 잘못된 주문을 삭제할 수 있다  
**So that** 주문 오류를 정정할 수 있다

**Acceptance Criteria:**
- [ ] 특정 주문을 삭제할 수 있다
- [ ] 삭제 전 확인 팝업이 표시된다
- [ ] 삭제 후 테이블 총 주문액이 재계산된다
- [ ] 삭제 성공/실패 피드백이 표시된다

---

## Epic 3: 테이블 세션 관리

### US-3.1: 테이블 초기 설정
**As a** 관리자  
**I want to** 테이블 태블릿을 초기 설정할 수 있다  
**So that** 고객이 해당 테이블에서 주문할 수 있다

**Acceptance Criteria:**
- [ ] 테이블 번호를 설정할 수 있다
- [ ] 테이블 비밀번호를 설정할 수 있다
- [ ] 설정 완료 후 자동 로그인이 활성화된다

---

### US-3.2: 테이블 이용 완료
**As a** 관리자  
**I want to** 고객 퇴장 시 테이블 이용 완료 처리를 할 수 있다  
**So that** 새 고객이 이전 주문 없이 시작할 수 있다

**Acceptance Criteria:**
- [ ] 이용 완료 버튼을 누르면 확인 팝업이 표시된다
- [ ] 이용 완료 시 해당 세션의 주문이 과거 이력으로 이동한다
- [ ] 이용 완료 시 테이블 주문 목록과 총액이 0으로 리셋된다
- [ ] 성공/실패 피드백이 표시된다

---

### US-3.3: 과거 주문 내역 조회
**As a** 관리자  
**I want to** 테이블의 과거 주문 내역을 조회할 수 있다  
**So that** 이전 고객의 주문 기록을 확인할 수 있다

**Acceptance Criteria:**
- [ ] 과거 내역 버튼으로 조회할 수 있다
- [ ] 테이블별 과거 주문이 시간 역순으로 표시된다
- [ ] 주문 번호, 시각, 메뉴, 금액, 이용 완료 시각이 표시된다
- [ ] 닫기 버튼으로 대시보드로 복귀할 수 있다

---

## Epic 4: 시스템 초기화 (MVP)

### US-4.1: 샘플 메뉴 데이터
**As a** 시스템  
**I want to** 시연용 샘플 메뉴가 자동 생성된다  
**So that** 메뉴 관리 기능 없이도 서비스를 시연할 수 있다

**Acceptance Criteria:**
- [ ] 시스템 시작 시 샘플 메뉴가 자동 생성된다
- [ ] 여러 카테고리의 메뉴가 포함된다
- [ ] 각 메뉴에 이름, 가격, 설명, 이미지 URL이 포함된다
