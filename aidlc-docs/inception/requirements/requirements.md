| 영역 | 선택 |
|-----|------|

## 3. 기능 요구사항 (Functional Requirements)

### 3.1 고객용 기능

| ID | 기능 | 설명 |
|----|-----|------|
| FR-C01 | 자동 로그인 | 테이블 태블릿 1회 설정 후 자동 인증 |
| FR-C02 | 메뉴 조회 | 카테고리별 메뉴 탐색, 상세 정보 표시 |
| FR-C03 | 장바구니 | 메뉴 추가/삭제, 수량 조절, 로컬 저장 |
| FR-C04 | 주문 생성 | 주문 확정 → 주문 번호 표시 → 메뉴 화면 리다이렉트 |
| FR-C05 | 주문 내역 | 현재 테이블 세션의 주문만 조회 |

### 3.2 관리자용 기능

| ID | 기능 | 설명 |
|----|-----|------|
| FR-A01 | 매장 인증 | 16시간 JWT 세션 |
| FR-A02 | 실시간 주문 모니터링 | SSE 기반, 그리드 레이아웃, 테이블별 카드 (총 주문액, 주문 시간 표시) |
| FR-A03 | 주문 상태 변경 | 주문접수/조리중/완료 상태를 버튼으로 선택 |
| FR-A04 | 지연 주문 강조 | 주문 후 30분 경과 시 카드 색상으로 구분 표시 |
| FR-A05 | 테이블 초기 설정 | 테이블 번호 및 비밀번호 설정 |
| FR-A06 | 주문 삭제 | 직권 삭제 기능 |
| FR-A07 | 매장 이용 완료 | 세션 종료 처리 |
| FR-A08 | 과거 주문 내역 | 테이블별 과거 주문 조회 |

### 3.3 데이터 초기화 (MVP)

| ID | 기능 | 설명 |
|----|-----|------|
| FR-D01 | 샘플 메뉴 Seed | 시연용 샘플 메뉴 데이터 자동 생성 |

## 4. 비기능 요구사항 (Non-Functional Requirements)

| ID | 영역 | 요구사항 |
|----|-----|---------|
| NFR-01 | 성능 | 메뉴 조회 응답 < 500ms |
| NFR-02 | 실시간성 | 신규 주문 2초 이내 관리자 화면 표시 (SSE) |
| NFR-03 | 보안 | bcrypt 비밀번호 해싱, JWT 인증 |
| NFR-04 | UI/UX | 터치 친화적 (최소 44x44px 버튼) |
| NFR-05 | 데이터 지속성 | 장바구니 로컬 저장 (새로고침 유지) |

---

## 5. 제외 범위 (Out of Scope)

### MVP 이후 (Backlog)
- 최신 주문 n개 미리보기
- 테이블별 필터링 기능
- 과거 주문 내역 날짜 필터링

### 프로젝트 범위 외

- 결제 처리 (PG 연동, 영수증, 환불, 포인트)
- 복잡한 인증 (OAuth, 2FA)
- 이미지 업로드/리사이징
- 알림 시스템 (푸시, SMS, 이메일)
- 주방 기능 (주방 디스플레이, 재고 관리)
- 분석/리포트 (매출 대시보드, 통계)
- 외부 연동 (배달 플랫폼, POS, 지도)
