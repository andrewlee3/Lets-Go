## 테스트 시나리오

### Scenario 1: 신규 사용자 (Happy Path)
1. ✅ `/table-setup` 접속
2. ✅ 매장 ID, 테이블 번호, 비밀번호 입력
3. ✅ 자동으로 `/` (메뉴 페이지) 이동
4. ✅ 카테고리 선택 → 메뉴 필터링
5. ✅ 메뉴 "담기" 버튼 → 장바구니에 추가
6. ✅ 장바구니 버튼 → Drawer 열림
7. ✅ 수량 조절 (+/-)
8. ✅ "주문하기" → `/cart` 페이지 이동
9. ✅ "주문 확정" → 주문 확인 모달
10. ✅ "주문 확정" → 주문 성공 화면 (5초)
11. ✅ 자동으로 `/` 이동
12. ✅ "주문내역" 버튼 → `/orders` 이동
13. ✅ 주문 내역 확인 (상태 뱃지)

### Scenario 2: 재방문 사용자
1. ✅ 브라우저 열기 → 자동 로그인
2. ✅ `/` (메뉴 페이지) 바로 표시
3. ✅ 이전 장바구니 데이터 유지 (있다면)

### Scenario 3: 장바구니 관리
1. ✅ 메뉴 추가 → 장바구니 개수 증가
2. ✅ 수량 조절 → 총 금액 실시간 업데이트
3. ✅ 항목 삭제 → 장바구니에서 제거
4. ✅ "전체 삭제" → 장바구니 비우기
5. ✅ 페이지 새로고침 → 장바구니 유지

### Scenario 4: 에러 케이스
1. ✅ 빈 장바구니로 주문 시도 → 알림 표시
2. ✅ 주문 실패 시 → 에러 메시지 표시
